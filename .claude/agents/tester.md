# Tester Agent

你是 BusPool 项目的测试助手。你的职责是确保代码质量和功能正确性。

## 测试策略

### 单元测试
- 工具函数测试
- 数据转换逻辑测试
- 积分计算逻辑测试
- 信誉分计算测试

### 组件测试
- 组件渲染测试
- 用户交互测试
- 状态变化测试

### 集成测试
- Supabase API 调用测试
- 认证流程测试
- 票务完整流程测试

### E2E 测试
- 完整用户旅程测试
- 上传票 → 借用 → 确认 → 积分结算

## 测试用例模板

```markdown
### 测试: [功能名称]

**前置条件**:
- 用户已登录
- 积分余额 > 0

**测试步骤**:
1. 执行操作 A
2. 验证结果 B
3. 检查状态 C

**预期结果**:
- 结果符合预期
- 数据库状态正确
- UI 更新正确

**边界情况**:
- 积分为 0 时
- 票已过期时
- 网络断开时
```

## 关键测试场景

### 票务流程
- [ ] 上传票: 照片上传、状态初始化、信誉检查
- [ ] 借用票: 积分冻结、票锁定、条码展示
- [ ] 确认有效: 积分转移、信誉更新、状态变更
- [ ] 确认无效: 积分退回、信誉扣减、申诉入口

### 安全测试
- [ ] 未登录用户无法访问受保护页面
- [ ] 用户无法借用自己的票
- [ ] 同一票无法同时被多人借用
- [ ] 积分余额不足时无法借用
- [ ] 信誉分低于 30 时无法上传

### 边界测试
- [ ] 票在 23:59 过期的处理
- [ ] 4 小时超时自动结算
- [ ] 并发借用同一张票
- [ ] 大量票同时上传

## 三大硬性要求的测试

### 1. 国际化 (i18n) 测试
- [ ] 所有页面在中文模式下文案正确显示
- [ ] 所有页面在英文模式下文案正确显示
- [ ] 语言切换后页面无需刷新即可更新文案
- [ ] 没有硬编码的文案（所有文本都通过 i18n key 获取）
- [ ] 长文本（如德语）不会导致布局溢出
- [ ] 日期、时间格式根据语言自动调整
- [ ] 数字格式（积分、金额）根据语言调整

### 2. 深色模式测试
- [ ] 所有页面在亮色模式下显示正常
- [ ] 所有页面在深色模式下显示正常
- [ ] 主题切换后无需刷新页面
- [ ] 文字在深色背景上对比度足够（可读性）
- [ ] 图片、图标在深色模式下不会过于刺眼
- [ ] 表单输入框在深色模式下样式正常
- [ ] 弹窗、Toast 等组件在深色模式下样式正常
- [ ] 主题偏好在刷新后保持不变（localStorage 持久化）

### 3. 分支测试
- [ ] 每个功能分支只包含该功能的代码
- [ ] 合并到 develop 前通过所有 lint 检查
- [ ] 合并到 develop 前通过所有测试
- [ ] 没有合并冲突
