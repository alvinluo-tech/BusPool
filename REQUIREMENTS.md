# BusPool — 公交日票共享平台 · 需求文档

## 一、项目背景

Durham 是英国一座大学城，学生日常出行依赖公交。一张 Day Ticket（Dayrider）售价 £1.6，允许当天无限次乘坐。但很多学生只用了几个小时就不再乘车，剩余价值被浪费。

**BusPool** 让这些闲置的 Day Ticket 流通起来：用完上午的学生将条码照片上传到平台，下午需要乘车的学生可以借用，实现"一人购票，全天复用"。

## 三、用户角色

| 角色 | 描述 |
|---|---|
| **上传者 (Seller)** | 买了 Day Ticket 但用不了一整天的学生，上传条码照片获取积分 |
| **借用者 (Borrower)** | 需要乘车的学生，用积分借用别人的条码 |
| **管理员 (Admin)** | 平台运营者，处理申诉、审核异常 |

## 四、核心页面与工作流

### 页面 1：首页（票务广场）

**功能**：展示当前可用的 Day Ticket 列表

**展示内容**：
- 每张票显示：上传者头像、上传时间、信誉分、历史成功率、所需积分
- 实时更新：有新票上传时无需刷新即可看到
- 筛选：按上传时间、信誉分排序
- 空状态：当没有可用票时，显示"暂无可用票，稍后再来"

**用户行为**：
- 浏览票列表 → 点击某张票 → 进入票详情

---

### 页面 2：票详情

**功能**：查看单张票的完整信息，决定是否借用

**展示内容**：
- 条码照片（模糊预览，借用后才显示清晰版）
- 上传者信息：头像、昵称、信誉分（0-100）、累计上传次数、历史成功率（如 95% 有效）
- 票信息：类型（Dayrider）、购买时间、有效区域
- 借用所需积分
- 倒计时：此票自动过期时间（当天 23:59）

**用户行为**：
- 点击"借用此票" → 确认弹窗 → 积分冻结 → 条码照片清晰展示 → 上车扫码

---

### 页面 3：上传票（发布闲置票）

**功能**：上传自己用不完的 Day Ticket

**流程**：
1. 拍摄或选择条码照片
2. 选择票类型（Dayrider / DaySaver）
3. 填写购买时间
4. 声明"本人今天已使用过此票"（勾选确认）
5. 提交上传

**限制**：
- 上传者信誉分必须 ≥ 30
- 同一票条码不能重复上传
- 每人每天最多上传 3 张票

**上传后**：
- 状态变为"待借用"
- 其他用户可在首页看到
- 被借用并确认有效后，上传者获得积分

---

### 页面 4：我的借用（借用记录）

**功能**：查看自己借过的票和当前正在使用的票

**状态流转**：
```
借用中 → 等待确认 → 已完成
         ↓
       已失效（票不能用）
```

**展示内容**：
- 正在使用中的票：条码照片（清晰）、借用时间、剩余有效时间
- 历史借用记录：票状态、是否有效、花费积分

**用户行为**：
- 查看当前票的条码 → 上车扫码 → 使用后点击"确认结果"

---

### 页面 5：确认结果（关键流程）

**功能**：借用者上车扫码后，确认票是否有效

**两种确认按钮**：
- ✅ **能用** — 票成功通过扫码器，正常乘车
- ❌ **不能用** — 票被扫码器拒绝，无法乘车

**确认"能用"后**：
- 冻结的积分转给上传者
- 上传者信誉 +2
- 交易完成

**确认"不能用"后**：
- 冻结的积分退回借用者
- 上传者信誉 -10
- 票标记为"已失效"
- 借用者可提交申诉（附照片等证据）

**超时规则**：
- 借用后 24 小时内未确认 → 系统自动按"能用"结算
- 借用后 2 小时内未确认 → 发送提醒通知

---

### 页面 6：积分钱包

**功能**：查看积分余额、收支明细

**展示内容**：
- 当前积分余额
- 积分明细列表：
  - 收入：上传的票被使用 +XX 积分
  - 支出：借用票 -XX 积分
  - 申诉退款 +XX 积分
- 积分兑换（未来可接入真实货币）

**积分定价参考**：
- 上传一张票被使用：+5 积分
- 借用一张票：-5 积分
- 新用户注册赠送：10 积分

---

### 页面 7：个人中心

**功能**：用户信息、信誉管理

**展示内容**：
- 头像、昵称、注册时间
- 信誉分（0-100）及等级标签
  - 80+ 优秀（绿色）
  - 50-79 良好（蓝色）
  - 30-49 一般（橙色）
  - 30 以下 限制（红色，禁止上传）
- 统计数据：上传次数、借用次数、成功率
- 信誉规则说明

**用户行为**：
- 编辑个人信息
- 查看信誉详情
- 查看申诉记录

---

### 页面 8：申诉中心

**功能**：当借用者确认"不能用"时，上传者可发起申诉

**流程**：
1. 上传者收到"票被确认无效"通知
2. 上传者可提交申诉：说明情况 + 证据（如自己当天使用照片）
3. 管理员审核
4. 裁定结果：维持原判 / 恢复信誉 / 恢复积分

---

## 五、通知系统

| 触发场景 | 通知内容 | 通知方式 |
|---|---|---|
| 有人借用你的票 | "你的票已被 XX 借用" | 站内 + 推送 |
| 借用的票即将过期 | "你借用的票将在 1 小时后过期，请尽快使用" | 推送 |
| 确认结果后 | "你获得 5 积分" / "你的票被确认无效" | 站内 |
| 申诉结果 | "申诉已处理，结果：XX" | 站内 |
| 新票可用 | "有新的 Dayrider 可用！" | 推送（可关闭） |

## 六、安全与防作弊机制

| 风险 | 应对 |
|---|---|
| 上传假票/旧票 | 信誉系统：假票导致信誉 -10，低于 30 禁止上传 |
| 借用者白嫖（说不能用但其实用了） | 信誉系统 + 24h 超时自动结算 |
| 同一票同时借给多人 | 票被借用后立即锁定，状态改为"使用中" |
| 条码照片被截图转发 | 借用期间才显示清晰版，过期后模糊处理 |
| 刷积分 | 同一票每天限制上传次数，异常交易人工审核 |

## 七、数据库核心表

```
users
├── id (UUID, auth.users)
├── email
├── nickname
├── avatar_url
├── points_balance (积分余额)
├── reputation (信誉分 0-100)
├── total_uploads (累计上传)
├── total_borrows (累计借用)
├── successful_uses (成功使用次数)
└── created_at

tickets
├── id (UUID)
├── uploader_id → users.id
├── barcode_image_url (Supabase Storage 路径)
├── ticket_type (dayrider / daysaver)
├── purchase_time (购买时间)
├── status (available / in_use / completed / expired / invalid)
├── zone (有效区域)
├── created_at
└── expires_at (当天 23:59)

transactions
├── id (UUID)
├── ticket_id → tickets.id
├── borrower_id → users.id
├── points_amount (积分数量)
├── status (pending / confirmed_valid / confirmed_invalid / auto_settled)
├── confirmed_at
├── created_at
└── expires_at (24h 自动结算时间)

appeals
├── id (UUID)
├── transaction_id → transactions.id
├── appellant_id → users.id
├── reason (申诉理由)
├── evidence_url (证据照片)
├── status (pending / resolved / rejected)
├── admin_note
└── created_at
```
